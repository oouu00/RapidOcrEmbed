# RapidOcrEmbed

[English](#english) | [中文](#中文)

<a id="english"></a>

> **NOTE** — This is a **derivative work** based on [RapidAI/RapidOcrOnnx](https://github.com/RapidAI/RapidOcrOnnx).
> It upgrades the OCR pipeline to **PaddleOCR PP-OCRv6**, adds an **embedded-model**
> build mode (**single-file DLL**, no external `.onnx` needed — hence the name *Embed*),
> and ships with pre-built **tiny / small / medium** Windows x64 DLLs.
> Licensed under the **Apache License 2.0**, same as the upstream project.

---

## ✨ What's new in this fork

| Area | Upstream (RapidOcrOnnx) | This project (RapidOcrEmbed) |
|------|------------------------|--------------|
| Models | PP-OCRv2/v3 | **PP-OCRv6** (det + rec) + **PP-LCNet** text-orientation classifier |
| Orientation (cls) | 0°/180° via small classifier | **PP-LCNet_x1_0_textline_ori**, input `[1,3,80,160]`, ImageNet normalization |
| Distribution | Separate `.onnx` + `.dll` | **Embedded mode** — models baked into the DLL as C headers → single file |
| Recognition dict | Classic `ppocr_keys` with `#`/space padding | **`ppocr_keys_v6.txt`** — first line is `blank`, no extra padding |
| Windows path | ANSI paths only | **UTF-8 / wide-char** path handling (Chinese paths supported) |
| Build | Dynamic deps | **Statically linked** ONNX Runtime + OpenCV, **static CRT (/MT)** |
| API | Basic C API | Extended: `OcrInitEmbedded`, `OcrDetectMem`, block-level getters |
| Table recognition | — | **SLANet** (deep learning) + **img2table** (pure OpenCV) + MinerU-style IoU matching → HTML |
| Inference engine | ONNX Runtime | **ONNX Runtime** (default) + **MNN** (lightweight alternative) |
| Variants | — | **tiny / small / medium** model sizes |

## 🚀 Quick start (use the pre-built DLL)

1. Download `RapidOcrOnnx_small.dll` (recommended) or `tiny`/`medium` from
   **[Releases](../../releases)**.
2. The DLL is **self-contained** — no external `.onnx` files required
   (models are embedded). Just place it next to your executable.
3. Load it via the C API or from any language that can call a DLL.
   See `测试/test_embedded.py` for a minimal Python example:

```python
import ctypes

dll = ctypes.CDLL("RapidOcrOnnx_small.dll")
dll.OcrInitEmbedded.restype = ctypes.c_void_p
handle = dll.OcrInitEmbedded(1)          # 1 = enable debug logging

# ... OCR_PARAM struct omitted, see 测试/ ...
dll.OcrDetectMem(handle, img_bytes, len(img_bytes), ctypes.byref(param))
length = dll.OcrGetLen(handle)
buf = ctypes.create_string_buffer(length + 1)
dll.OcrGetResult(handle, buf, length + 1)
print(buf.value.decode("utf-8"))

dll.OcrDestroy(handle)
```

## 🔧 Build from source

### Requirements

- **Visual Studio 2019 (v142)** or newer with C++ workload
- **CMake ≥ 3.17**
- **Python 3.8+** with `pyyaml` (for the embed step)
- Static **ONNX Runtime** and **OpenCV** packages (see Releases)

### Steps

```bash
# 1. Put the static deps in place
#    onnxruntime-static/windows-x64/   (contains include/ + lib/)
#    opencv-static/windows-x64/        (OpenCV static build)

# 2. (Optional) Generate embedded-model headers from your .onnx
python "模型转二进制/embed_models.py"
#    → writes include/models/model_det.h, model_cls.h, model_rec.h, model_keys.h

# 3. Build all three variants (tiny / small / medium)
python build.py
#    → produces output/RapidOcrOnnx_{tiny,small,medium}.{dll,lib}
```

### CMake options

| Option | Values | Default | Description |
|--------|--------|---------|-------------|
| `OCR_OUTPUT` | `BIN` / `CLIB` / `JNI` | `BIN` | Executable / **DLL+LIB** / Java binding |
| `OCR_ONNX` | `CPU` / `CUDA` | `CPU` | Inference provider |
| `OCR_EMBEDDED_MODELS` | `ON` / `OFF` | `OFF` | Bake models into the binary |
| `OCR_BUILD_CRT` | `True` / `False` | — | Use static CRT `/MT` |

## 📖 C API reference

All functions are exported with the `Ocr*` prefix (`extern "C"`). Key entry points:

| Function | Purpose |
|----------|---------|
| `OcrInitEmbedded(nThreads)` | Create a handle using the **embedded** models (no file paths) |
| `OcrInit(det, cls, rec, keys, nThreads)` | Create a handle loading models from disk |
| `OcrDetect(handle, dir, name, param)` | Detect via **file path** |
| `OcrDetectMem(handle, bytes, size, param)` | Detect via **in-memory image bytes** |
| `OcrGetLen / OcrGetResult` | Read the concatenated text result |
| `OcrDetectEx / OcrDetectMemEx` | Detect and keep **per-block** results |
| `OcrGetBlockCount / OcrGetBlockText / OcrGetBlockScore / OcrGetBlockBox / OcrGetBlockAngle / OcrGetBlockCharScores` | Inspect each text block |
| `OcrDestroy(handle)` | Release the handle |
| `OcrInitTableEmbedded(nThreads)` | Initialize **OCR + SLANet** for table recognition (embedded models) |
| `OcrDetectTable(handle, dir, name, param)` | Detect table via file path → returns HTML with text |
| `OcrDetectTableMem(handle, bytes, size, param)` | Detect table via in-memory image bytes |
| `OcrGetTableLen / OcrGetTableResult` | Read the HTML table result |
| `OcrGetTableStructureScore` | Get table structure recognition confidence |
| `OcrGetTableOcrText` | Get concatenated OCR text (no cell association) |

Full signatures are in [`include/OcrLiteCApi.h`](include/OcrLiteCApi.h).

### `OCR_PARAM` struct

```c
typedef struct __ocr_param {
    int   padding;         // default 50
    int   maxSideLen;      // default 1024 (longest side after resize)
    float boxScoreThresh;  // default 0.6
    float boxThresh;       // default 0.3
    float unClipRatio;     // default 2.0
    int   doAngle;         // 1 = run orientation classifier
    int   mostAngle;       // 1 = ...
} OCR_PARAM;
```

# 中文说明

> **说明** — 本项目是 [RapidAI/RapidOcrOnnx](https://github.com/RapidAI/RapidOcrOnnx) 的**派生修改版本**。
> 将 OCR 流水线升级到 **PaddleOCR PP-OCRv6**，新增**模型内嵌**编译模式（**单一 DLL、无需外部 `.onnx`**——故名 *Embed*），
> 并提供 **tiny / small / medium** 三种预编译 Windows x64 DLL。
> 许可证沿用上游 **Apache License 2.0**。

## ✨ 相比上游的改动

- **模型升级到 PP-OCRv6**：检测模型 (det) + 识别模型 (rec)
- **方向分类器**：改用 `PP-LCNet_x1_0_textline_ori`，输入 `[1,3,80,160]`，ImageNet 归一化，输出 0°/180°
- **模型内嵌模式**：把 `.onnx` / 字典文件转成 C 头文件并编进 DLL，**单文件分发**，运行时不再需要任何模型文件
- **字典处理**：采用 `ppocr_keys_v6.txt`，首行为 `blank`，不再插入 `#`/空格
- **Windows 中文路径**：全面使用 UTF-8 / 宽字符 API，支持中文路径
- **静态链接**：ONNX Runtime + OpenCV 全部静态链接，CRT 使用 `/MT`，生成的 DLL 可独立分发
- **扩展 C API**：新增 `OcrInitEmbedded`、`OcrDetectMem`（内存图片）、逐文本块查询接口等
- **表格识别**：支持两种模式 —— **SLANet**（深度学习模型）+ **img2table**（纯 OpenCV 有线框表格）+ MinerU 风格 IoU 文本匹配 → 输出含文本的 HTML 表格
- **推理引擎**：默认 **ONNX Runtime**，可选 **MNN**（阿里巴巴轻量级推理引擎）
- **三种规格**：tiny / small / medium，按需取舍体积与精度

1. 从 **[Releases](../../releases)** 下载 `RapidOcrOnnx_small.dll`（推荐）或 tiny/medium。

print(buf.value.decode("utf-8"))

## 📖 C API 接口

所有导出函数以 `Ocr*` 为前缀（`extern "C"`）。主要入口：

| 函数 | 作用 |
|------|------|
| `OcrInitEmbedded(nThreads)` | 使用**内嵌模型**创建句柄（无需文件路径） |
| `OcrInit(det, cls, rec, keys, nThreads)` | 从磁盘加载模型创建句柄 |
| `OcrDetect(handle, dir, name, param)` | 按**文件路径**识别 |
| `OcrDetectMem(handle, bytes, size, param)` | 按**内存图片字节**识别 |
| `OcrGetLen / OcrGetResult` | 读取拼接后的纯文本结果 |
| `OcrDetectEx / OcrDetectMemEx` | 识别并保留**逐文本块**结果 |
| `OcrGetBlockCount / OcrGetBlockText / OcrGetBlockScore / OcrGetBlockBox / OcrGetBlockAngle / OcrGetBlockCharScores` | 查询每个文本块 |
| `OcrDestroy(handle)` | 释放句柄 |
| `OcrInitTableEmbedded(nThreads)` | 初始化 **OCR + SLANet** 用于表格识别（内嵌模型） |
| `OcrDetectTable(handle, dir, name, param)` | 文件路径方式检测表格 → 返回含文本的 HTML |
| `OcrDetectTableMem(handle, bytes, size, param)` | 内存图片方式检测表格 |
| `OcrGetTableLen / OcrGetTableResult` | 读取表格 HTML 结果 |
| `OcrGetTableStructureScore` | 获取表格结构识别置信度 |
| `OcrGetTableOcrText` | 获取拼接后的 OCR 文本（无单元格关联） |

完整签名见 [`include/OcrLiteCApi.h`](include/OcrLiteCApi.h)。
