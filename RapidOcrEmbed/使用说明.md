# RapidOcrOnnx DLL 使用说明

## 一、DLL/LIB 参数配置

### 1.1 OCR_PARAM 参数结构体

DLL 通过 `OCR_PARAM` 结构体传递识别参数，定义如下：

```c
typedef struct __ocr_param {
    int padding;          // 图像边距填充，默认 50
    int maxSideLen;       // 最大边长限制，默认 1024
    float boxScoreThresh; // 文本框置信度阈值，默认 0.6
    float boxThresh;      // 文本框阈值，默认 0.3
    float unClipRatio;    // 文本框扩展比例，默认 2.0
    int doAngle;          // 是否启用方向分类，默认 1（开启）
    int mostAngle;        // 是否使用多数投票确定角度，默认 1（开启）
} OCR_PARAM;
```

### 1.2 参数详细说明

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `padding` | int | 50 | 图像边缘填充像素数。增加边距可以检测到边缘附近的文本。 |
| `maxSideLen` | int | 1024 | 图像缩放后的最大边长。超过此值的图像会被缩小，加快处理速度。设为 0 表示不限制。 |
| `boxScoreThresh` | float | 0.6 | 文本框置信度阈值。值越低检测到的文本框越多，但可能包含误检。 |
| `boxThresh` | float | 0.3 | 文本框二值化阈值。影响文本区域的分割效果。 |
| `unClipRatio` | float | 2.0 | 文本框扩展比例。值越大文本框范围越宽，适合识别长文本。 |
| `doAngle` | int | 1 | **方向分类开关**。1=启用，0=禁用。启用后会自动检测文本行方向并旋转校正。 |
| `mostAngle` | int | 1 | 多数投票开关。当多个文本行方向不一致时，采用多数投票确定整体方向。 |

### 1.3 参数调整建议

**场景一：检测更多文本**
```python
param = OCR_PARAM(
    padding=50,
    maxSideLen=960,
    boxScoreThresh=0.3,   # 降低阈值
    boxThresh=0.2,        # 降低阈值
    unClipRatio=1.5,
    doAngle=1,
    mostAngle=1
)
```

**场景二：高精度识别**
```python
param = OCR_PARAM(
    padding=50,
    maxSideLen=1024,
    boxScoreThresh=0.7,   # 提高阈值
    boxThresh=0.4,        # 提高阈值
    unClipRatio=2.0,
    doAngle=1,
    mostAngle=1
)
```

**场景三：禁用方向分类（已知文本方向正确）**
```python
param = OCR_PARAM(
    padding=50,
    maxSideLen=1024,
    boxScoreThresh=0.6,
    boxThresh=0.3,
    unClipRatio=2.0,
    doAngle=0,            # 禁用方向分类
    mostAngle=0
)
```

## 二、方向分类功能详解

### 2.1 文本行方向分类（已实现）

当前 DLL 内置的 `AngleNet`（使用 `PP-LCNet_x1_0_textline_ori_infer.onnx`）用于检测**每个文本行**的方向：

- **输入**：裁剪后的文本行图像，尺寸 `[1, 3, 80, 160]`
- **输出**：方向分类结果
  - `0` = 正常方向（0°）
  - `1` = 旋转 180°
- **处理流程**：
  1. 检测文本区域（det 模型）
  2. 裁剪每个文本行
  3. 对每个文本行进行方向分类
  4. 如果检测到 180°，自动旋转文本行
  5. 进行文字识别（rec 模型）

**默认状态**：`doAngle=1`，方向分类**默认开启**。

### 2.2 整图方向分类（新增功能）

`ch_ppocr_mobile_v2.0_cls_train.onnx` 用于检测**整张图片**的方向：

- **用途**：在 OCR 处理前，先判断图片整体方向（0°/90°/180°/270°）
- **处理流程**：
  1. 对整张图片进行方向分类
  2. 根据分类结果旋转图片到正确方向
  3. 将旋转后的图片交给 OCR 进行识别

**适用场景**：
- 手机拍摄的照片可能存在各种旋转角度
- 扫描仪扫描的文档可能方向不正确
- 批量处理大量图片时自动校正方向


## 四、Python 调用示例

### 4.1 基础使用

```python
import ctypes
import os

# 定义参数结构体
class OCR_PARAM(ctypes.Structure):
    _fields_ = [
        ("padding", ctypes.c_int),
        ("maxSideLen", ctypes.c_int),
        ("boxScoreThresh", ctypes.c_float),
        ("boxThresh", ctypes.c_float),
        ("unClipRatio", ctypes.c_float),
        ("doAngle", ctypes.c_int),
        ("mostAngle", ctypes.c_int),
    ]

# 加载 DLL
dll = ctypes.CDLL("RapidOcrOnnx_small.dll")

# 设置函数签名
dll.OcrInitEmbedded.restype = ctypes.c_void_p
dll.OcrInitEmbedded.argtypes = [ctypes.c_int]

dll.OcrDetectMem.restype = ctypes.c_bool
dll.OcrDetectMem.argtypes = [
    ctypes.c_void_p,
    ctypes.POINTER(ctypes.c_ubyte),
    ctypes.c_int,
    ctypes.POINTER(OCR_PARAM)
]

dll.OcrGetLen.restype = ctypes.c_int
dll.OcrGetLen.argtypes = [ctypes.c_void_p]

dll.OcrGetResult.restype = ctypes.c_int
dll.OcrGetResult.argtypes = [ctypes.c_void_p, ctypes.c_char_p, ctypes.c_int]

dll.OcrDestroy.restype = None
dll.OcrDestroy.argtypes = [ctypes.c_void_p]

# 初始化（使用内嵌模型）
handle = dll.OcrInitEmbedded(4)  # 4 线程

# 读取图片
with open("test.jpg", "rb") as f:
    img_data = f.read()

img_bytes = (ctypes.c_ubyte * len(img_data))(*img_data)

# 设置参数（默认参数）
param = OCR_PARAM(
    padding=50,
    maxSideLen=1024,
    boxScoreThresh=0.6,
    boxThresh=0.3,
    unClipRatio=2.0,
    doAngle=1,      # 开启方向分类
    mostAngle=1
)

# 识别
success = dll.OcrDetectMem(handle, img_bytes, len(img_data), ctypes.byref(param))

if success:
    # 获取结果
    length = dll.OcrGetLen(handle)
    buffer = ctypes.create_string_buffer(length + 1)
    dll.OcrGetResult(handle, buffer, length + 1)
    result = buffer.value.decode("utf-8")
    print(f"识别结果: {result}")

# 销毁句柄
dll.OcrDestroy(handle)
```

### 4.2 获取详细结果（逐文本块）

```python
# 设置扩展函数签名
dll.OcrDetectMemEx.restype = ctypes.c_int
dll.OcrDetectMemEx.argtypes = [
    ctypes.c_void_p,
    ctypes.POINTER(ctypes.c_ubyte),
    ctypes.c_int,
    ctypes.POINTER(OCR_PARAM)
]

dll.OcrGetBlockCount.restype = ctypes.c_int
dll.OcrGetBlockCount.argtypes = [ctypes.c_void_p]

dll.OcrGetBlockText.restype = ctypes.c_int
dll.OcrGetBlockText.argtypes = [ctypes.c_void_p, ctypes.c_int, ctypes.c_char_p, ctypes.c_int]

dll.OcrGetBlockScore.restype = ctypes.c_float
dll.OcrGetBlockScore.argtypes = [ctypes.c_void_p, ctypes.c_int]

dll.OcrGetBlockBox.restype = ctypes.c_int
dll.OcrGetBlockBox.argtypes = [ctypes.c_void_p, ctypes.c_int, ctypes.POINTER(ctypes.c_int)]

dll.OcrGetBlockAngle.restype = ctypes.c_int
dll.OcrGetBlockAngle.argtypes = [ctypes.c_void_p, ctypes.c_int]

# 识别并获取文本块数量
block_count = dll.OcrDetectMemEx(handle, img_bytes, len(img_data), ctypes.byref(param))

print(f"检测到 {block_count} 个文本块")

# 遍历每个文本块
for i in range(block_count):
    # 获取文本
    text_len = dll.OcrGetBlockText(handle, i, None, 0)
    text_buf = ctypes.create_string_buffer(text_len + 1)
    dll.OcrGetBlockText(handle, i, text_buf, text_len + 1)
    text = text_buf.value.decode("utf-8")
    
    # 获取置信度
    score = dll.OcrGetBlockScore(handle, i)
    
    # 获取坐标框
    box = (ctypes.c_int * 8)()
    dll.OcrGetBlockBox(handle, i, box)
    coords = [(box[j*2], box[j*2+1]) for j in range(4)]
    
    # 获取方向
    angle = dll.OcrGetBlockAngle(handle, i)
    
    print(f"块 {i}: 文本='{text}', 置信度={score:.2f}, 方向={angle}°")
    print(f"  坐标: {coords}")
```

## 六、常见问题

### Q1: 方向分类默认是否开启？
**答：是的，默认开启。** `doAngle` 默认值为 `1`，会自动检测每个文本行的方向并校正。

### Q2: 如何禁用方向分类？
**答：设置 `doAngle=0`。** 如果已知图片方向正确，禁用可提高处理速度。

### Q3: 方向分类能处理哪些角度？
**答：当前仅支持 0° 和 180°。** 文本行方向分类器输出为二分类（正常/旋转180°）。

### Q4: 如何处理 90°/270° 旋转的图片？
**答：需要使用整图方向分类模型。** 将 `ch_ppocr_mobile_v2.0_cls_train.onnx` 模型嵌入后，可在 OCR 前先判断整图方向并旋转。

### Q5: 参数调整后识别效果变差怎么办？
**答：建议使用默认参数。** 默认参数经过优化，适用于大多数场景。特殊场景可微调：
- 检测不到文本：降低 `boxScoreThresh` 和 `boxThresh`
- 检测到太多噪声：提高阈值
- 文本被截断：增大 `unClipRatio`

## 七、性能优化建议

1. **线程数设置**：`OcrInitEmbedded(nThreads)` 参数建议设置为 CPU 核心数
2. **图像尺寸**：较小的图像处理更快，可适当降低 `maxSideLen`
3. **禁用方向分类**：已知方向正确时禁用 `doAngle` 可节省约 10% 时间
4. **批量处理**：复用 OCR 句柄，避免重复初始化

## 十、表格识别功能（新增）

### 10.1 概述

DLL 内嵌 **SLANet** 表格结构识别模型 + **img2table** 纯 OpenCV 算法 + MinerU 风格的 IoU 文本匹配，支持两种表格识别模式：

| 模式 | 算法 | 特点 | 适用场景 |
|------|------|------|----------|
| **模式 1 (SLANet)** | 深度学习模型 | 精度高，支持复杂表格结构 | 无线框表格、合并单元格、复杂排版 |
| **模式 2 (img2table)** | 纯 OpenCV | 无需模型，速度快，仅支持有线框表格 | 规整的线框表格、简单表格 |

**工作流程**：

**SLANet 模式（模式 1）**：
1. OCR 检测图片中所有文本块 → 获取文本内容与坐标
2. SLANet 模型识别表格结构 → 输出单元格坐标与 HTML 骨架
3. IoU 匹配 → 将 OCR 文本填入对应单元格
4. HTML 后处理 → 处理 `<eb>` 特殊 token、表头加粗等

**img2table 模式（模式 2）**：
1. OCR 检测图片中所有文本块 → 获取文本内容与坐标
2. OpenCV 霍夫线检测 → 提取表格线框结构
3. 基于线框构建单元格 → 输出 HTML 骨架
4. IoU 匹配 → 将 OCR 文本填入对应单元格

### 10.2 表格识别 API

```c
// 初始化（OCR + SLANet 内嵌模型一并加载）
OCR_HANDLE OcrInitTableEmbedded(int nThreads);

// 设置表格模式（0=SLANet, 1=img2table）
void OcrSetTableMode(OCR_HANDLE handle, int mode);

// 检测表格（文件路径方式）
OCR_BOOL OcrDetectTable(
    OCR_HANDLE handle,
    const char *imgPath,      // 图片目录
    const char *imgName,      // 图片文件名
    OCR_PARAM *pParam
);

// 检测表格（内存图片方式）
OCR_BOOL OcrDetectTableMem(
    OCR_HANDLE handle,
    const unsigned char *imgData,
    int imgSize,
    OCR_PARAM *pParam
);

// 获取 HTML 结果长度
int OcrGetTableLen(OCR_HANDLE handle);

// 获取 HTML 结果（已包含文本）
int OcrGetTableResult(OCR_HANDLE handle, char *szBuf, int nLen);

// 获取结构识别置信度
float OcrGetTableStructureScore(OCR_HANDLE handle);

// 获取纯 OCR 文本（无单元格关联，冗余信息）
int OcrGetTableOcrText(OCR_HANDLE handle, char *szBuf, int nLen);
```

**表格模式说明**：
- `OcrSetTableMode(handle, 0)`：SLANet 模式（深度学习，支持复杂表格）
- `OcrSetTableMode(handle, 1)`：img2table 模式（纯 OpenCV，仅支持有线框表格）

### 10.3 CLI 命令行用法

```bash
# SLANet 模式（默认，推荐）
OcrQtCpp.exe --cli --table=1 表格图片.jpg

# img2table 模式（纯 OpenCV，仅支持有线框表格）
OcrQtCpp.exe --cli --table=2 表格图片.jpg

# 多文件表格识别
OcrQtCpp.exe --cli --table=1 表格1.jpg 表格2.jpg

# 表格 + 坐标组合
OcrQtCpp.exe --cli --table=1 --coords 表格图片.jpg
```

**输出格式**：
```json
{
  "code": 0,
  "type": "table",
  "text": "表格中的纯文本内容...",
  "html": "<table><tr><td>...</td></tr></table>",
  "structureScore": 0.99
}
```

**注意事项**：
- 使用 `--table=1` 或 `--table=2` 格式，避免参数解析问题
- 表格识别会同时生成 `.html` 文件（同名）
- img2table 模式仅适用于有清晰线框的表格，不支持合并单元格

### 10.6 开源算法说明

**MinerU IoU 匹配算法**

DLL 内部的 IoU 匹配算法移植自 [MinerU](https://github.com/opendatalab/MinerU) 3.4.0：

| 组件 | 源文件 | 说明 |
|------|--------|------|
| `calcMineruDist()` | matcher_utils.py `distance()` | 四角差之和 + min(左上角差, 右下角差) |
| IoU 计算 | matcher_utils.py `compute_iou()` | 标准 Intersection over Union |
| `_filter_ocr_result` | matcher.py `_filter_ocr_result()` | 过滤表格顶线以上的文本 |
| 匹配循环 | matcher.py `match_result()` + `_select_best_cell_indices()` | IoU 优先，相同时距离排序，阈值 1e-8 |

**img2table 线框表格检测算法**

模式 2 的表格线框检测算法移植自 [xavctn/img2table](https://github.com/xavctn/img2table)：

| 组件 | 说明 |
|------|------|
| 霍夫线检测 | OpenCV `HoughLinesP` 提取表格线框 |
| 线框合并 | 基于距离和角度合并相近线段 |
| 单元格构建 | 根据横纵线交点构建单元格坐标 |
| 表格结构 | 输出 HTML 骨架（无合并单元格支持） |
